Nostalgic Gaming

I recently had a hankering to play some Final Fantasy VII and I actually found the game on the Apple App Store for $14.99 USD. I bought it immediatly for my iPad just to relive all the nostalgia from my childhood.

About 15 minutes into the game, my hands started cramping and I was getting frustrated by the controls. I kept running into objects, exiting rooms, moving in directions I never intended. It was awkward and annoying. To be honest, I missed the entire experience of having a controller and playing it on my TV.

Enter OpenEmu

OpenEmu is a great open source project that finally brings game emulations to the OS X. Before I got rid of my PC, I was able to play all these games and find emulators for them rather easily. On the Mac however, its actually quite difficult and somewhat of a challenge. Thanks to the folks at OpenEmu, those headaches are now all gone!

Installation: Cloning or Website

Theres a two different ways to install OpenEmu, I won't go to deep into the steps here but I'll list the links for you and let you decide for yourself which installation type is right for your needs.

  • Download from Website I recommend using the experimental build, it has more support for more consoles, also this is the easiest/fastest way to install OpenEmu
  • Compile from Source I tried this method out and compiled the code in X-Code but I honestly see no difference from this version and the experimental version I downloaded off the website

Compiling the source code off github is for advanced users. You'll need authorized SSH keys, a GitHub Account and the latest version of X-Code installed before you proceed

Setup

Once installed, launch OpenEmu, you'll be greeted with some fancy splash screens before you're taken to your library:

Drag and drop your .cue files into the library. The app will begin scanning the file and searching for the associated .bin files:

For games like FFVII that span multiple discs, you'll want to create a plain text .m3u file. To do this, first load all the .cue files into your library. When the library is done scanning, open up the text editor of your choice, I used Sublime Text 3. Copy the .cue files into the text editor, making sure you save the file as PLAIN TEXT with the extension .m3u:

Instead of launching the individual disc files, running the .m3u files allows you to switch discs in the game rather than having to back out to the library:

Troubleshooting:

If you're lucky and your build somehow included the following .bin files, then congratulations, go play the game and stop reading this none sense. However if you're unlucky like me and got the below error message, download this file: OpenEmu-2-BIOS-pack. It includes all the latest bios you'll need to get some of the emulators working. To install, simply unzip the file and drag the necessary .bin files into your library like you did with the .cue files.

Controller Support:

OpenEmu has a lot of support for many different types of controllers, however, I had none of the supported controllers on the list. I was lucky enough to find support for my Xbox One controller though. Just install this little gem from Xbox 360 Controller on GitHub, the latest release provides Xbox One controller support. Please note that you'll need a USB cable hooked up to the controller as the bluetooth technology in the controller is proprietary and not universal like the PS4 controllers.
